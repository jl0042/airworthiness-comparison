把 169 条逐条对比看完, 真正站得住的判断只有几条:

一 章节骨架直接复用, 数字往下调

UMR / UPL 用了 CCAR-23 2022 修订时引入的新四位编号体系 (23.2230 → UMR.2230 / UPL.2230), 169 条里有 38 条话题完全相同。结构、载荷、性能、稳定性这些不分载人不载人的条款都保留, 真正在变的是数字 (例如限制载荷系数因为不载人风险结构往下降)。

二 删除的章节加起来比想象中少, 新增的章节加起来比想象中多

「整章删除」只有 13 条 (CCAR-23 60 多条里大约 1/5), 主要集中在乘员保护 + 驾驶舱仪表 + 老 H 章电推。但「新增」有 102 条 (H 章电动发动机 40 + I 章螺旋桨 21 + L 章 C2 链路 5 + R 章遥控台 21 + F 章 DAA / 围栏 / 网安 4 + 其他 11)。不载人不等于减负, 而是把保护乘员的预算搬到保护第三方 + 系统软件 + 链路 + 地面站。

三 真正首次出现的是软件和链路

CCAR-23 / 27 里几乎没有对「链路 / 自主性 / 网络安保 / 软件围栏」的硬性要求 — 因为飞行员就在机内, 这些不是适航关心的事。UMR / UPL 把 L 章 (C2 链路) + F 章新增 (DAA / 电子围栏 / 网络安保) 共 8 条提到设备级 / 系统级。这是无人机适航相对有人机最显眼的新东西, 也是低空经济商业化的真实门槛 — 没有飞行员兜底, 这些项一项失效就是不可控事故。

四 CCAR-25 / CCAR-29 的运输类加严, UMR / UPL 没复制

运输类的两件标志性事 — 25.803 / 29.803 的 90 秒撤离演示, 25.571 / 29.571 的全机疲劳试验, 25.1309 / 29.1309 的双 / 三冗余 — UMR / UPL 都没复制过去。UMR / UPL 是「正常类」无人机, 它们的对位是 CCAR-23 / 27, 不是 25 / 29。重型 (≥5700 kg 固定翼, ≥3180 kg 旋翼) 无人机目前在中国还没有专门标准, 走的是专用条件路径 (例如峰飞 V2000CG 2 吨级走 SC, 亿航 EH216-S 载客 eVTOL 走 SC-21-002)。
